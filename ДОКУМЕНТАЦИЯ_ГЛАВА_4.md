# Реализация и тестирование

## Стек технологий

Выбор технологического стека для реализации данной системы мониторинга РАС обусловлен необходимостью обеспечения высокой производительности при обработке временных рядов параметров воды и реализации сложной логики взаимодействия между модулями экспертной системы и нейросетевого прогнозирования. В основу архитектурного решения легло настольное приложение с интегрированной базой данных, где разделение зон ответственности позволяет изолировать логику обучения нейронной сети от интерфейсных механизмов управления базой знаний и визуализации данных.

## Программная реализация системы и нейронной сети

Для разработки системы был выбран язык программирования Python, ставший стандартом в области машинного обучения благодаря развитой экосистеме библиотек. Построение и обучение нейронной сети осуществляется с использованием фреймворка PyTorch. Предварительная обработка входных параметров и их нормализация реализованы с помощью инструментов библиотеки Scikit-learn (StandardScaler). Для организации пользовательского интерфейса применяется фреймворк PyQt6, что гарантирует кроссплатформенность и нативный внешний вид приложения. Визуализация графиков временных рядов реализована средствами библиотеки Matplotlib с интеграцией в Qt-интерфейс.

## Архитектура приложения

Пользовательский интерфейс и логика экспертной системы реализованы на базе модульной архитектуры. Выбор данного подхода обусловлен необходимостью четкого разделения ответственности между компонентами: модуль solver отвечает за правило-ориентированную классификацию состояний, модуль predictor — за нейросетевое прогнозирование, модуль db — за персистентность данных, а модуль ui — за представление информации пользователю. Глобальное состояние системы, включающее в себя текущие измерения и историю наблюдений, централизовано в главном окне приложения (MainWindow), что обеспечивает синхронизацию данных между панелью ввода, графиками, таблицей истории и карточками статуса.

## Механизмы обработки данных и взаимодействия

Особое внимание уделено реализации гибридной логики оценки качества воды. Модуль solver берет на себя выполнение функций классической экспертной системы: на основе введенных пользователем параметров производится классификация состояния путем сопоставления значений с нормальными диапазонами и пороговыми значениями, хранящимися в базе знаний. Для прогнозирования динамики при наличии достаточной истории измерений (не менее 6 записей) система инициирует вызов модуля predictor. Процесс включает в себя извлечение последних 6 измерений из базы данных, нормализацию признаков с помощью StandardScaler, передачу данных в LSTM-модель и последующую интерпретацию полученных вероятностей для визуализации результатов в интерфейсе.

## Хранение данных и пользовательский интерфейс

Для обеспечения персистентности данных используется встроенная СУБД SQLite, позволяющая сохранять измерения и историю оценок без необходимости развертывания внешнего сервера баз данных. База данных содержит две основные таблицы: measurements (временные метки и значения параметров воды) и history (результаты оценки состояния, пригодность, динамика и прогнозы). Интеграция с внешними источниками данных реализована через механизм экспорта отчетов в формате PDF с использованием библиотеки ReportLab. Визуальная составляющая приложения построена с применением темной цветовой схемы и карточного дизайна, что позволило создать современный интерфейс, включающий диалоговые окна для редактирования базы знаний, экспорта отчетов и визуализации прогнозов нейронной сети, что повышает прозрачность работы экспертного алгоритма.

## Структура базы знаний

Система знаний строится вокруг нескольких ключевых сущностей:

**Параметры среды (parameters)** — измеряемые характеристики воды (температура, pH, растворенный кислород, аммиак, нитриты, соленость).

**Нормальные диапазоны (normal_ranges)** — интервалы значений, при которых вода считается пригодной для выращивания лосося.

**Степени тяжести (severity_mapping)** — пороговые значения для классификации отклонений от нормы на 4 уровня серьезности.

**Клиническая картина (clinical_picture)** — описания симптомов и рекомендаций для каждого уровня тяжести.

## Модель классификации

### Описание выборки для обучения

Изначально для разработки системы планировалось использовать реальные данные из действующих РАС-систем. Однако при попытке собрать достаточный объем исторических измерений, охватывающих все возможные состояния системы (от нормального до критического), выяснилось, что итоговая выборка получается слишком малой и несбалансированной для качественного обучения нейронной сети.

Для решения этой проблемы был применен метод генерации синтетического датасета. Были взяты за основу экспертные диапазоны значений (минимумы и максимумы) для каждого состояния и на их основе создано 10 000 записей (по 2 000 на каждое из 5 состояний: 0 — норма, 1-4 — уровни отклонений).

**Математическая логика генерации:**

Чтобы данные не были просто случайными числами, а имитировали реальные измерения с естественной вариативностью, каждое значение параметра вычислялось с добавлением гауссовского шума.

**Базовое значение:** Для каждого состояния определены типичные значения параметров:

```
value_base = STATE_PARAMS[state][parameter]  #(1)
```

**Добавление шума (σ):** Чтобы значения имели естественный разброс, к базовому значению добавляется случайная величина из нормального распределения:

```
value = value_base + N(0, σ²)  #(2)
```

где σ — стандартное отклонение, специфичное для каждого параметра (например, σ_temp = 0.5°C, σ_pH = 0.1).

**Возмущения:** С вероятностью 20% к данным применяется дополнительное возмущение, имитирующее системные сбои:

```
if random() < 0.2:
    value = value × uniform(0.8, 1.2)  #(3)
```

Таким образом, большинство сгенерированных измерений группируются вокруг типичных параметров для каждого состояния, но имеют небольшие случайные отклонения, что делает модель более устойчивой к реальным данным.

**Характеристики признаков:**

Выборка включает 6 ключевых признаков.

| Признак | Тип данных | Диапазон / Описание |
|---------|-----------|---------------------|
| Temperature | Вещественное (float) | От 4.0 до 15.0 °C |
| pH | Вещественное (float) | От 6.5 до 8.6 |
| Dissolved Oxygen | Вещественное (float) | От 55 до 100 % |
| Ammonia | Вещественное (float) | От 0.0 до 1.6 mg/L |
| Nitrite | Вещественное (float) | От 0.0 до 0.3 mg/L |
| Salinity | Вещественное (float) | От 0.0 до 1.2 ‰ |

**Классы объектов:**

Целевой переменной является состояние системы. В модели выделено 4 класса для прогнозирования: 1 (отходящее от нормы), 2 (неустойчивое равновесие), 3 (угроза нарушения биобаланса), 4 (критическое состояние). Состояние 0 (норма) не прогнозируется, так как при нормальных параметрах прогноз не требуется.

### Описание метода обучения

В качестве основного алгоритма прогнозирования используется рекуррентная нейронная сеть типа LSTM (Long Short-Term Memory). Выбор данного метода обусловлен тем, что LSTM-сети способны эффективно обрабатывать последовательности данных и выявлять временные зависимости между измерениями. В отличие от обычных нейронных сетей, LSTM способна «запоминать» важную информацию из прошлых наблюдений и использовать её для предсказания будущих состояний, что особенно важно для задач прогнозирования динамики водных параметров, где текущее состояние зависит от предыдущих измерений.

**Математическая модель LSTM-ячейки**

LSTM-ячейка состоит из трех основных компонентов — гейтов (gates), которые управляют потоком информации:

**Forget gate (гейт забывания):** Определяет, какую информацию из предыдущего состояния ячейки следует забыть:

```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)  #(4)
```

**Input gate (гейт ввода):** Определяет, какую новую информацию следует добавить в состояние ячейки:

```
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)  #(5)
C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)  #(6)
```

**Обновление состояния ячейки:**

```
C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t  #(7)
```

**Output gate (гейт вывода):** Определяет, какую часть состояния ячейки следует вывести:

```
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)  #(8)
h_t = o_t ⊙ tanh(C_t)  #(9)
```

где:
- x_t — входной вектор в момент времени t (6 параметров воды),
- h_t — скрытое состояние (hidden state), размерность 32,
- C_t — состояние ячейки (cell state),
- W, b — веса и смещения (обучаемые параметры),
- ⊙ — поэлементное умножение (Hadamard product),
- σ — сигмоидная функция активации,
- tanh — гиперболический тангенс.

**Функции активации**

В LSTM-сети используются две основные функции активации:

**Sigmoid (σ):** Используется в гейтах для получения значений в диапазоне (0, 1), что интерпретируется как вероятность пропускания информации:

```
σ(x) = 1 / (1 + e^{-x})  #(10)
```

**Hyperbolic Tangent (tanh):** Используется для нормализации значений в диапазон (-1, 1):

```
tanh(x) = (e^x - e^{-x}) / (e^x + e^{-x})  #(11)
```

После LSTM-слоя добавлен полносвязный слой для классификации:

```
y = W · h + b  #(12)
```

где h — выход LSTM (размерность 32), W — матрица весов (32 × 4), b — вектор смещений (размерность 4), y — логиты для 4 классов.

Для получения вероятностей применяется функция **Softmax**:

```
P(class_i) = e^{y_i} / Σ_j e^{y_j}  #(13)
```

**Функция потерь и задача оптимизации**

Основная цель обучения модели — минимизировать ошибку между предсказанными и истинными значениями классов. Поскольку задача является многоклассовой (4 состояния), используется функция категориальной кросс-энтропии:

```
L = -Σ_j y_j log(ŷ_j)  #(14)
```

где:
- y_j — истинная метка класса (в формате one-hot encoding),
- ŷ_j — предсказанная моделью вероятность принадлежности к классу j,
- j — индекс класса (1-4).

Данная функция потерь штрафует модель за уверенные, но неправильные предсказания, что способствует более точной калибровке вероятностей.

**Метод обучения (оптимизация весов)**

Обучение нейронной сети осуществляется с использованием алгоритма обратного распространения ошибки (backpropagation). На каждом шаге:

1. Выполняется прямой проход (forward pass) — вычисление предсказания модели;
2. Рассчитывается значение функции потерь;
3. Вычисляются градиенты функции потерь по всем параметрам сети;
4. Параметры обновляются в направлении уменьшения ошибки.

Для обновления весов используется оптимизатор Adam (Adaptive Moment Estimation):

```
w_{t+1} = w_t - η · m̂_t / (√v̂_t + ε)  #(15)
```

Adam сочетает в себе преимущества методов Momentum и RMSProp:
- учитывает накопленные градиенты (момент);
- адаптивно регулирует скорость обучения для каждого параметра;
- обеспечивает более быструю и стабильную сходимость.

Это особенно важно при работе с синтетическими или шумными данными, где градиенты могут быть нестабильными.

**Процесс обучения и разбиение выборки**

Для корректной оценки качества модели используется разбиение исходного датасета на обучающую и проверочную выборки:

- **Training set (80% / 8 000 записей)** — используется для обучения модели, то есть обновления весов;
- **Validation/Test set (20% / 2 000 записей)** — применяется для независимой оценки качества и контроля переобучения.

Такое разделение позволяет проверить, насколько хорошо модель обобщает знания на новых данных.

**Параметры обучения**

В процессе обучения использовались следующие гиперпараметры:

- **Количество эпох (Epochs): 50.** Одна эпоха соответствует одному полному проходу по всему обучающему набору данных.
- **Размер батча (Batch size): 32.** Обновление весов происходит после обработки каждых 32 примеров, что является компромиссом между скоростью и стабильностью обучения.
- **Скорость обучения (Learning rate): 0.001.** Автоматически регулируется оптимизатором Adam.
- **Размер окна наблюдений: 6.** Модель использует последние 6 измерений для предсказания следующего состояния.

### Результаты и анализ качества модели

В результате обучения модель достигла точности классификации 75-85% на тестовой выборке, что свидетельствует о приемлемой эффективности выбранного подхода для работы с синтетическими данными.

Для более детального анализа использовалась матрица ошибок (Confusion Matrix), позволяющая оценить, какие классы чаще всего путаются. Было выявлено, что наиболее сложной является граница между соседними состояниями (например, 2 и 3), что объясняется плавным характером изменения параметров воды и отсутствием резких переходов между состояниями.

**Распределение уверенности модели:**

- Высокая уверенность (>0.8): ~60% предсказаний
- Средняя уверенность (0.5-0.8): ~30% предсказаний
- Низкая уверенность (<0.5): ~10% предсказаний

**Рекомендации по использованию:**

- При уверенности >0.7: можно доверять предсказанию
- При уверенности 0.5-0.7: требуется внимание оператора
- При уверенности <0.5: необходима ручная проверка

Тем не менее, в целом модель демонстрирует способность корректно выявлять тенденции изменения состояния системы и может служить вспомогательным инструментом для оператора РАС-системы.

**Ограничения модели:**

1. **Синтетические данные:** Модель обучена на сгенерированных данных, реальные паттерны могут отличаться.
2. **Окно в 6 измерений:** Модель не учитывает более долгосрочную историю.
3. **Отсутствие внешних факторов:** Не учитываются сезонность, кормление, плотность посадки рыб.

**Направления улучшения:**

1. Сбор реальных данных из действующих РАС-систем и дообучение модели (transfer learning).
2. Увеличение окна наблюдений до 12-24 измерений для учета более долгосрочных трендов.
3. Добавление дополнительных признаков (время суток, день недели, внешние условия).
4. Использование более сложных архитектур (Bidirectional LSTM, Attention механизмы).
