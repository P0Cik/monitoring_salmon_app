# RAS Monitor - Система мониторинга водной среды УЗВ

Гибридная система мониторинга для установок замкнутого водоснабжения (УЗВ) на основе:
- **Жёсткого решателя** (по правилам) для определения текущего состояния
- **Нейросети LSTM** для прогнозирования состояния на шаг вперёд (t+1)

## 📁 Структура проекта

```
ras_monitor/
├── main.py                 # Точка входа, инициализация QApplication

├── ml/
│   ├── __init__.py

    └── main_window.py      # PyQt6 интерфейс
```

### Уровни тяжести состояний (1→4)
1. **Отходящее от нормы**
2. **Неустойчивое равновесие**
3. **Угроза нарушения биобаланса**
4. **Критическое состояние**

**Температура:**
- [11;12)∪(14;15] → 1

- [7.2;7.4)∪(7.8;8.0] → 1
- [7.0;7.2)∪(8.0;8.2] → 2
- [6.6;7.0)∪(8.2;8.6] → 3
- <6.6 или >8.6 → 4

**O₂:**
- [85;90) → 1
- [70;85) → 2
- [60;70) → 3
- <60 → 4

**Аммиак:**
- [0.5;0.6) → 1
- [0.6;1.0) → 2
- [1.0;1.5) → 3
- ≥1.5 → 4

**Нитриты:**
- [0.1;0.15) → 1
- [0.15;0.2) → 2
- [0.2;0.25) → 3
- ≥0.25 → 4

**Солёность:**
- (0;0.1) → 1
- [0.1;0.5) → 2

## 🚀 Пошаговая инструкция запуска

### Шаг 1: Установка зависимостей

```bash
cd ras_monitor
pip install -r requirements.txt
```

### Шаг 2: Обучение нейросети (генерация model.pth и scaler.pkl)

```bash
python ml/train.py
```

Процесс обучения:
- Генерирует 10 000 синтетических временных рядов
- Добавляет шум ±3% к параметрам
- Обучает LSTM (1 скрытый слой, 6 входов, 4 выхода)
- Сохраняет `model.pth` и `scaler.pkl` в папку `ml/`

### Шаг 3: Запуск приложения

```bash
python main.py
```

Откроется главное окно с:
- **Слева**: форма ввода 6 параметров + кнопка «Добавить замер»
- **Центр**: график временных рядов всех параметров
- **Справа**: карточки состояния, пригодности, динамики и прогноза
- **Внизу**: таблица истории замеров

### Шаг 4: Упаковка в .exe (для Windows)

```bash
python build.py
```

Результат:
- `dist/RAS Monitor/` — папка с исполняемым файлом и всеми зависимостями
- `RAS Monitor.exe` — готовый к запуску файл

**Важно:** Используется режим `--onedir` (не `--onefile`), так как PyTorch требует загрузки множества DLL-библиотек.

## 🧠 Архитектура системы

### Жёсткий решатель (solver.py)

Алгоритм 3.c.i:
1. Если ВСЕ параметры в нормальных диапазонах → оценка = «пригодна», состояние = 0
2. Иначе → для каждого параметра найти уровень тяжести (1-4), итоговое состояние = MAX(уровней), оценка = «непригодна»
3. Динамика (ретроспективная): сравнить текущее состояние с предыдущим из БД
   - Равно → «Стабильно»
   - Текущее > предыдущего → «Ухудшение»
   - Текущее < предыдущего → «Улучшение»

### Нейросеть (LSTM)

- **Вход**: окно из последних 6 замеров (shape: 1×6×6)
- **Архитектура**: 
  - LSTM: 1 слой, 32 скрытых единицы
  - Dense: 32 → 4 выхода (состояния 1-4)
  - Softmax для вероятностей
- **Выход**: вероятности состояний {1: p₁, 2: p₂, 3: p₃, 4: p₄}
- **Прогноз динамики**: если argmax(probs) > current_state → «Ухудшение» и т.д.

### База данных (SQLite)

**Таблица `measurements`:**
- id, timestamp, temp, ph, o2, ammonia, nitrite, salinity

**Таблица `history`:**
- id, measurement_id, current_state, suitability, past_dynamics, forecast_dynamics, forecast_confidence

## 📝 Примечания

1. **PyInstaller и PyTorch**: Модель упаковывается через `--add-data`, все torch-библиотеки собираются через `--collect-all=torch`.

2. **sys._MEIPASS**: Для корректной работы в собранном .exe пути к model.pth/scaler.pkl резолвятся через `sys._MEIPASS`.

3. **Синтетические данные**: При обучении генерируются реалистичные временные ряды с постепенными переходами между состояниями.

4. **Окно прогноза**: Для работы нейросети требуется минимум 6 сохранённых замеров в БД.
