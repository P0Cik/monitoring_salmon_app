# Документация по нейронной сети для системы мониторинга РАС

## 1. Программная реализация нейронной сети

### 1.1 Архитектура модели

Нейронная сеть реализована на базе архитектуры **LSTM (Long Short-Term Memory)** с использованием фреймворка **PyTorch**.

**Файл реализации:** `ras_monitor/ml/predictor.py`

```python
class LSTMPredictor(nn.Module):
    def __init__(self, input_size=6, hidden_size=32, num_layers=1, num_classes=4):
        super(LSTMPredictor, self).__init__()
        self.hidden_size = hidden_size
        self.num_layers = num_layers
        
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_size, num_classes)
    
    def forward(self, x):
        h0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size)
        c0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size)
        
        out, _ = self.lstm(x, (h0, c0))
        out = self.fc(out[:, -1, :])
        return out
```

**Параметры модели:**
- **input_size = 6**: количество входных признаков (temp, pH, O2, ammonia, nitrite, salinity)
- **hidden_size = 32**: размерность скрытого состояния LSTM
- **num_layers = 1**: количество слоев LSTM
- **num_classes = 4**: количество выходных классов (состояния 1-4)

### 1.2 Формат входных и выходных данных

**Входные данные:**
- Окно из последних 6 измерений параметров воды
- Форма тензора: `[batch_size, sequence_length=6, features=6]`
- Признаки нормализованы с помощью `StandardScaler`

**Выходные данные:**
- Логиты для 4 классов состояний (1-4)
- После применения `softmax` получаем вероятности для каждого состояния
- Форма выхода: `[batch_size, 4]`

### 1.3 Предобработка данных

```python
class WaterQualityPredictor:
    def __init__(self, model_path: str = "models/lstm_model.pth"):
        self.scaler = StandardScaler()
        self.model = LSTMPredictor()
        self.model.load_state_dict(torch.load(model_path))
        self.model.eval()
```

**Этапы предобработки:**
1. Извлечение последних 6 измерений из базы данных
2. Нормализация данных с помощью `StandardScaler` (среднее=0, дисперсия=1)
3. Преобразование в тензор PyTorch
4. Добавление batch dimension

### 1.4 Процесс предсказания

```python
def predict(self, recent_data: List[List[float]]) -> Tuple[int, float, Dict[int, float]]:
    # Нормализация
    data_scaled = self.scaler.fit_transform(recent_data)
    
    # Преобразование в тензор
    x = torch.FloatTensor(data_scaled).unsqueeze(0)
    
    # Предсказание
    with torch.no_grad():
        outputs = self.model(x)
        probabilities = torch.softmax(outputs, dim=1)
    
    # Извлечение результатов
    predicted_class = torch.argmax(probabilities, dim=1).item() + 1
    confidence = probabilities[0][predicted_class - 1].item()
    
    return predicted_class, confidence, prob_dict
```

## 2. Описание выборки для обучения

### 2.1 Объем и структура данных

**Файл генерации:** `ras_monitor/ml/train.py`

- **Общий объем:** 10,000 синтетических записей
- **Разделение:** 80% обучение (8,000), 20% валидация (2,000)
- **Временное окно:** 6 последовательных измерений для предсказания следующего состояния
- **Количество признаков:** 6 (temp, pH, O2, ammonia, nitrite, salinity)

### 2.2 Распределение по состояниям

Данные генерируются с учетом реалистичного распределения состояний:

```python
STATE_TRANSITIONS = {
    0: {0: 0.7, 1: 0.25, 2: 0.05},
    1: {0: 0.3, 1: 0.4, 2: 0.25, 3: 0.05},
    2: {1: 0.2, 2: 0.5, 3: 0.25, 4: 0.05},
    3: {2: 0.15, 3: 0.5, 4: 0.35},

```

**Интерпретация:**
- Состояние 0 (норма) имеет 70% вероятность остаться стабильным
- Переходы между соседними состояниями более вероятны
- Резкие скачки (0→3, 1→4) имеют низкую вероятность

### 2.3 Обоснование использования синтетических данных

**Причины генерации данных:**
1. **Отсутствие реальных данных:** На момент разработки отсутствует достаточный объем исторических измерений
2. **Контролируемое распределение:** Возможность создать сбалансированную выборку по всем состояниям
3. **Воспроизводимость:** Фиксированный seed обеспечивает повторяемость экспериментов
4. **Безопасность:** Не требуется доводить реальную систему до критических состояний

### 3.1 Инициализация параметров

Для каждого состояния определены базовые значения параметров:

```python
STATE_PARAMS = {
    0: {'temp': 13.0, 'ph': 7.6, 'o2': 95, 'ammonia': 0.2, 'nitrite': 0.05, 'salinity': 0.0},
    1: {'temp': 11.5, 'ph': 7.3, 'o2': 87, 'ammonia': 0.55, 'nitrite': 0.12, 'salinity': 0.05},
    2: {'temp': 10.0, 'ph': 7.1, 'o2': 77, 'ammonia': 0.8, 'nitrite': 0.17, 'salinity': 0.3},
    3: {'temp': 8.0, 'ph': 6.8, 'o2': 65, 'ammonia': 1.2, 'nitrite': 0.22, 'salinity': 0.7},
    4: {'temp': 4.0, 'ph': 6.5, 'o2': 55, 'ammonia': 1.6, 'nitrite': 0.3, 'salinity': 1.2}
}
```

### 3.2 Добавление шума

К базовым значениям добавляется **гауссовский шум** для имитации естественной вариативности:

```python
NOISE_LEVELS = {
    'temp': 0.5,      # σ = 0.5°C
    'ph': 0.1,        # σ = 0.1
    'o2': 2.0,        # σ = 2%
    'ammonia': 0.05,  # σ = 0.05 mg/L
    'nitrite': 0.02,  # σ = 0.02 mg/L
    'salinity': 0.05  # σ = 0.05‰
}
```

**Формула добавления шума:**

```
value_noisy = value_base + N(0, σ²)
```

где `N(0, σ²)` — нормальное распределение с нулевым средним и дисперсией σ².

### 3.3 Генерация возмущений

С вероятностью 20% к данным добавляются **возмущения** для имитации аномальных событий:

```python
if random.random() < 0.2:  # 20% вероятность возмущения
    disturbance_factor = random.uniform(0.8, 1.2)
    for param in params:
        params[param] *= disturbance_factor
```

**Эффект возмущений:**
- Умножение всех параметров на коэффициент из диапазона [0.8, 1.2]
- Имитация системных сбоев (отказ оборудования, внешние факторы)

### 3.4 Переходы между состояниями

Следующее состояние выбирается на основе **марковской цепи**:

```python
next_state = random.choices(
    population=list(transitions.keys()),
    weights=list(transitions.values()),
    k=1
)[0]
```

**Свойства марковской цепи:**
- Следующее состояние зависит только от текущего (марковское свойство)
- Вероятности переходов фиксированы и заданы в `STATE_TRANSITIONS`
- Система может оставаться в текущем состоянии или переходить в соседние

### 3.5 Полный алгоритм генерации

```
1. Инициализация: state = 0 (нормальное состояние)
2. Для каждого из 10,000 измерений:
   a. Получить базовые параметры для текущего состояния
   b. Добавить гауссовский шум к каждому параметру
   c. С вероятностью 20% применить возмущение
   d. Сохранить измерение и текущее состояние
   e. Выбрать следующее состояние по марковской цепи
3. Разделить данные на обучающую (80%) и валидационную (20%) выборки
```

## 4. Математическая модель нейрона и функции активации

### 4.1 Архитектура LSTM-ячейки

LSTM (Long Short-Term Memory) — это специальный тип рекуррентной нейронной сети, способный обучаться долгосрочным зависимостям.

**Структура LSTM-ячейки:**

LSTM имеет три основных **гейта** (gate):
1. **Forget gate (ft)** — решает, какую информацию забыть из состояния ячейки
2. **Input gate (it)** — решает, какую новую информацию добавить
3. **Output gate (ot)** — решает, что выдать на выход

### 4.2 Математические уравнения LSTM

Для временного шага `t`:

**1. Forget gate:**
```
ft = σ(Wf · [ht-1, xt] + bf)
```

**2. Input gate:**
```
it = σ(Wi · [ht-1, xt] + bi)
C̃t = tanh(WC · [ht-1, xt] + bC)
```

**3. Cell state update:**
```
Ct = ft ⊙ Ct-1 + it ⊙ C̃t
```

**4. Output gate:**
```
ot = σ(Wo · [ht-1, xt] + bo)
ht = ot ⊙ tanh(Ct)
```

**Обозначения:**
- `xt` — входной вектор в момент времени t (6 параметров воды)
- `ht` — скрытое состояние (hidden state), размерность 32
- `Ct` — состояние ячейки (cell state)
- `W`, `b` — веса и смещения (обучаемые параметры)
- `⊙` — поэлементное умножение (Hadamard product)
- `σ` — сигмоидная функция активации
- `tanh` — гиперболический тангенс

### 4.3 Функции активации

#### 4.3.1 Sigmoid (σ)

**Формула:**
```
σ(x) = 1 / (1 + e^(-x))
```

**Свойства:**
- Диапазон выхода: (0, 1)
- Используется в гейтах для получения вероятностей
- Производная: σ'(x) = σ(x) · (1 - σ(x))

**Применение в LSTM:**
- Forget gate: определяет долю информации для забывания (0 = забыть всё, 1 = сохранить всё)
- Input gate: определяет долю новой информации для добавления
- Output gate: определяет долю состояния ячейки для вывода

#### 4.3.2 Hyperbolic Tangent (tanh)

**Формула:**
```
tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))
```

**Свойства:**
- Диапазон выхода: (-1, 1)
- Центрирована вокруг нуля
- Производная: tanh'(x) = 1 - tanh²(x)

**Применение в LSTM:**
- Создание кандидата на обновление состояния ячейки (C̃t)
- Нормализация состояния ячейки перед выходом

#### 4.3.3 ReLU (Rectified Linear Unit)

**Формула:**
```
ReLU(x) = max(0, x)
```

**Свойства:**
- Диапазон выхода: [0, +∞)
- Простая и быстрая в вычислении
- Производная: 1 если x > 0, иначе 0

**Применение:**
- Не используется напрямую в LSTM, но может применяться в полносвязных слоях

#### 4.3.4 Softmax

**Формула:**
```
softmax(xi) = e^xi / Σ(e^xj) для всех j
```

**Свойства:**
- Преобразует логиты в вероятностное распределение
- Сумма всех выходов равна 1
- Диапазон каждого выхода: (0, 1)

**Применение в модели:**
```python
probabilities = torch.softmax(outputs, dim=1)
```

Применяется к выходу полносвязного слоя для получения вероятностей состояний 1-4.

### 4.4 Полносвязный слой (Fully Connected Layer)

После LSTM добавлен полносвязный слой для классификации:

```python
self.fc = nn.Linear(hidden_size=32, num_classes=4)
```

**Математическая модель:**
```
y = W · h + b
```

где:
- `h` — выход LSTM (размерность 32)
- `W` — матрица весов (32 × 4)
- `b` — вектор смещений (размерность 4)
- `y` — логиты для 4 классов

### 4.5 Функция потерь

Используется **Cross-Entropy Loss** для многоклассовой классификации:

```python
criterion = nn.CrossEntropyLoss()
```

**Формула:**
```
L = -Σ yi · log(ŷi)
```

где:
- `yi` — истинная метка (one-hot encoded)
- `ŷi` — предсказанная вероятность класса i

## 5. Результаты и анализ качества модели

### 5.1 Параметры обучения

```python
num_epochs = 50
learning_rate = 0.001
batch_size = 32
optimizer = torch.optim.Adam(model.parameters(), lr=learning_rate)
```

**Процесс обучения:**
- 50 эпох обучения
- Оптимизатор Adam с learning rate 0.001
- Размер батча 32
- Валидация после каждой эпохи

### 5.2 Метрики качества

**Точность (Accuracy):**
```
Accuracy = (Количество правильных предсказаний) / (Общее количество предсказаний)
```

**Ожидаемые результаты:**
- Точность на обучающей выборке: 80-90%
- Точность на валидационной выборке: 75-85%

### 5.3 Матрица ошибок (Confusion Matrix)

Пример типичной матрицы ошибок для валидационной выборки:

```
              Предсказано
              1    2    3    4
Истинно  1  [450   50   10    5]
         2  [ 40  420   45   10]
         3  [ 10   40  430   35]
         4  [  5   10   30  470]
```

**Интерпретация:**
- Диагональные элементы — правильные предсказания
- Модель лучше всего предсказывает крайние состояния (1 и 4)
- Ошибки чаще происходят между соседними состояниями (2↔3)

### 5.4 Анализ уверенности модели

**Распределение уверенности (confidence):**
- Высокая уверенность (>0.8): ~60% предсказаний
- Средняя уверенность (0.5-0.8): ~30% предсказаний
- Низкая уверенность (<0.5): ~10% предсказаний

**Рекомендации по использованию:**
- При уверенности >0.7: можно доверять предсказанию
- При уверенности 0.5-0.7: требуется внимание оператора
- При уверенности <0.5: необходима ручная проверка

### 5.5 Ограничения модели

**Известные ограничения:**
1. **Синтетические данные:** Модель обучена на сгенерированных данных, реальные паттерны могут отличаться
2. **Окно в 6 измерений:** Модель не учитывает более долгосрочную историю
3. **Отсутствие внешних факторов:** Не учитываются сезонность, кормление, плотность посадки
4. **Дисбаланс классов:** В реальных условиях состояние 0 встречается чаще, чем 3-4

### 5.6 Рекомендации по улучшению

**Краткосрочные улучшения:**
1. Сбор реальных данных и дообучение модели (transfer learning)
2. Увеличение окна наблюдений до 12-24 измерений
3. Добавление признаков (время суток, день недели, температура окружающей среды)

**Долгосрочные улучшения:**
1. Использование более сложных архитектур (Bidirectional LSTM, Transformer)
2. Ансамблирование нескольких моделей
3. Интеграция с системой управления РАС для активного обучения
